# Case Study 1 – Windows Discovery Activity

## Scenario

During routine monitoring, Wazuh generated an alert indicating that a Windows Discovery command had been executed on the monitored endpoint.

The objective of this investigation was to determine whether the activity represented legitimate administrative behavior or a potential security threat.

---

## Alert Summary

| Field | Value |
|--------|-------|
| Timestamp | Jul 9, 2026 @ 20:07:53 |
| Host | DESKTOP-MARCIO |
| User | Marcio-Braga\yukem |
| Event ID | 1 (Process Creation) |
| Rule ID | 92031 |
| Rule Description | Discovery activity executed |
| Severity | 3 |
| MITRE ATT&CK | T1087 – Account Discovery |

---

## Initial Evidence

The Wazuh alert indicated the execution of a Windows Discovery command.

The investigation identified the following process:

```

Process: net1.exe

Command:

net localgroup administrators

```

This command is commonly used to enumerate members of the local Administrators group on a Windows system.

## Investigation

The investigation focused on identifying the origin of the process and determining whether the execution was expected.

### Process Created

```

C:\Windows\System32\net1.exe

```

### Parent Process

```

C:\Windows\System32\net.exe

```

### Executed Command

```

net localgroup administrators

```

### User

```

Marcio-Braga\yukem

```

### Process Purpose

The command enumerates all members of the local **Administrators** group.

This activity is commonly observed during system administration tasks but is also frequently used by attackers during the **Discovery** phase after gaining access to a host.

---

## MITRE ATT&CK Analysis

| Tactic | Technique | ID |
|---------|-----------|----|
| Discovery | Account Discovery | T1087 |

The generated alert was correctly mapped to the **MITRE ATT&CK T1087 – Account Discovery** technique, since the executed command attempts to identify privileged local accounts.

---

## Analyst Assessment

The investigation confirmed that the command was executed manually by the legitimate user **Marcio-Braga\yukem** during a controlled laboratory exercise.

No additional suspicious activity, privilege escalation, persistence mechanisms, or malicious processes were identified.

Although the command is frequently observed during post-exploitation activities, its execution alone is insufficient to classify the event as malicious.

---

## Analyst Verdict

| Result | Classification |
|----------|---------------|

| ❌ Security Incident | Not confirmed |

---

## Lessons Learned

This investigation demonstrates that not every alert generated by a SIEM indicates malicious activity.

Commands such as **net localgroup administrators** are legitimate Windows administrative utilities but are also commonly used during adversary reconnaissance. Therefore, SOC analysts must evaluate the execution context, user, parent process, and surrounding activity before determining whether escalation is necessary.
